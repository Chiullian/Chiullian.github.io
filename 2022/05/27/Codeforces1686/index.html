[Everything Everywhere All But One]

You are given an array of nnn integers a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​. After you watched the amazing film “Everything Everywhere All At Once”, you came up with the following operation.In one operation, you choose n−1n-1n−1 elements of the array and replace each of them with their arithmetic mean (which doesn’t have to be an integer). For example, from the array [1,2,3,1][1, 2, 3, 1][1,2,3,1] we can get the array [2,2,2,1][2, 2, 2, 1][2,2,2,1], if we choose the first three elements, or we can get the array [43,43,3,43][\frac{4}{3}, \frac{4}{3}, 3, \frac{4}{3}][34​,34​,3,34​], if we choose all elements except the third.Is it possible to make all elements of the array equal by performing a finite number of such operations?

n长度的数组选n-1个变成他们的平均值,问是否存在能使他们全部相同的变法

思路:

枚举每一位即可,看是否正确,范围很小,暴力即可

[Odd Subarrays]

​ For an array [b1,b2,…,bm][b_1, b_2, \ldots, b_m][b1​,b2​,…,bm​] define its number of inversions as the number of pairs (i,j)(i, j)(i,j) of integers such that 1≤i<j≤m1 \le i < j \le m1≤i<j≤m and bi>bjb_i>b_jbi​>bj​. Let’s call array bbb odd if its number of inversions is odd.

​ For example, array [4,2,7][4, 2, 7][4,2,7] is odd, as its number of inversions is 111, while array [2,1,4,3][2, 1, 4, 3][2,1,4,3] isn’t, as its number of inversions is 222.You are given a permutation [p1,p2,…,pn][p_1, p_2, \ldots, p_n][p1​,p2​,…,pn​] of integers from 111 to nnn (each of them appears exactly once in the permutation).

​ You want to split it into several consecutive subarrays (maybe just one), so that the number of the odd subarrays among them is as large as possible. What largest number of these subarrays may be odd?

贪心即可,肯定是越短的越好,每次只取最短的即可,从前往后遍历

[Circular Local MiniMax]

​ You are given nnn integers a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​. Is it possible to arrange them on a circle so that each number is strictly greater than both its neighbors or strictly smaller than both its neighbors?In other words, check if there exists a rearrangement b1,b2,…,bnb_1, b_2, \ldots, b_nb1​,b2​,…,bn​ of the integers a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ such that for each iii from 111 to nnn at least one of the following conditions holds:

  • bi−1<bi>bi+1b_{i-1} < b_i > b_{i+1}bi−1​<bi​>bi+1​
  • bi−1>bi<bi+1b_{i-1} > b_i < b_{i+1}bi−1​>bi​<bi+1​

​ To make sense of the previous formulas for i=1i=1i=1 and i=ni=ni=n, one shall define b0=bnb_0=b_nb0​=bn​ and bn+1=b1b_{n+1}=b_1bn+1​=b1​.

题意:随便排列一个序列,满足bi−1<bi>bi+1b_{i-1} < b_i > b_{i+1}bi−1​<bi​>bi+1​, 和bi−1>bi<bi+1b_{i-1} > b_i < b_{i+1}bi−1​>bi​<bi+1​

思路:

直接按照原序列排个序即可,从前往后一个隔一个先放一半, 然后在从前往后接着放,如果这种都不满足,那么一定就是不存在的,因为我们已经尽可能的考虑用小的去匹配小的了

[Linguistics]

​ Alina has discovered a weird language, which contains only 444 words: A\texttt{A}A, B\texttt{B}B, AB\texttt{AB}AB, BA\texttt{BA}BA. It also turned out that there are no spaces in this language: a sentence is written by just concatenating its words into a single string.

​ Alina has found one such sentence sss and she is curious: is it possible that it consists of precisely aaa words A\texttt{A}A, bbb words B\texttt{B}B, ccc words AB\texttt{AB}AB, and ddd words BA\texttt{BA}BA?

​ In other words, determine, if it’s possible to concatenate these a+b+c+da+b+c+da+b+c+d words in some order so that the resulting string is sss. Each of the a+b+c+da+b+c+da+b+c+d words must be used exactly once in the concatenation, but you can choose the order in which they are concatenated.

题意

给出若干个A,B,AB,BA,再给出一个字符串,判断该字符串能否由给出的这些字符拼接而成.

不会做T_T, 这里分享一个在知乎大佬的题解思路 大佬的题解链接

首先我们用双指针枚举出每一段相邻字符都不等的子串,形如ABABA, ABAB, BABA, BABAB.

以最少需要多少个A和AB为例.B和BA可以用类似的方法算出.

我们发现长度为偶数len的这种子串,只有两种可能分别是以A开头的形如ABAB,或者以B开头形如BABA对于以B开头的子串,显然并一定不需要AB,而以A开头的子串,如果我们还有单独的一对A和B的碎片剩余,那么我们就可以通过消耗一对A和B的碎片删去子串头和尾的字符把这个子串变成以B开头,这样就不需要AB这种碎片了,但是如果我们没有一对A和B的碎片剩余,那么就至少需要len/2个AB碎片.所以我们可以记录所有的长度为偶数,假设我们还剩下r对单独的A和B碎片,那么我们就可以删掉长度最长的r个子串,剩下的子串长度之和就是最少需要的AB串数量.而对于至少需要的A或B串数量,偶数长度串是不会产生贡献的.

而对于长度为奇数的子串,我们可以选择是删掉头还是删掉尾来使这个子串变成ABAB形还是BABA形,所以我们这种串只会用掉一个A或B碎片但是不会对最少需要的AB产生贡献.

得到了最少需要的A,B, AB,BA这些碎片最少需要的数量后判断拥有的数量是否大于最少需要的数量即可.
